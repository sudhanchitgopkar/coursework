02.02.21

Reviewing δ

DFAs

  • δ: Q × Σ → Q
  • input: (state, symbol)
  • output: state

NFAs

  • Not allowing ε transitions
    • δ: Q × Σ → \powerset(Q)
    • input: (state, symbol)
    • output: set of states
  • If an automaton is nondeterministic, then the codomain of δ is a power set
  • Allowing ε transitions,
    • δ: Q × (\Simga ∪ {ε}) → \powerset(Q)
    • input: (state, symbol or ε)
    • output: set of states
  • Note that ε is not a symbol and cannot belong to Σ
    • This is because symbols have a length of 1 and ε, by definition, has a length of 0

NFA

  • General 5-tuple including
    1. Q: set of states
    2. Σ - input alphabet
    3. δ: Q × (Σ ∪ {ε}) → \powerstate(Q)
    4. q0 ∈ Q - starting state
    5. F ⊂ Q - set of final states
  • There may exist some inputs for which it is possible that the NFA accepts the sequence if one path is taken and rejects the sequence if another path is taken
  • The NFA does not have the ability to look ahead at possible states

Computation

  • The NFA runs all possible branches on a given string simultaneously and independently
  • A string, w, is rejected by an NFA, N, if every branch of the nondeterminism tree for N on w rejects
    • ¬ ∃ a branch that accepts
  • A string, w, is accepted by an NFA, N, if ∃ a branch on the nondeterminism tree for which N on w accepts
  • Theorem: Any language recognized by an NFA can be recognized by a DFA
    • Let N be an NFA.
    • Then ∃ a DFA, M, with L(M) = L(N)
    • The language recognized by a machine, M, (L(M)) is the set of exactly all strings accepted by M (no rejected strings allowed)
      • ¬ L(M) = set of all strings rejected by M

Closure Introduction

  • The language recognized by a DFA, M, (L(M)) is the set of all string accepted by M
  • Thus, M = (Q, Σ, δ, q0,F)
  • And Σ* is the universe of all possible inputs to M
  • ∀ strings w ∈ Σ*, M either accepts or rejects w
  • It follows that L(M) ⊂ Σ*
  • And ¬ L(M) = Σ* - L(M)
  • Therefore, M accepts every string in L(M) and rejects everything in ¬ L(M)

Closure Continued

  • A set, A, is closed under a binary operation, OP, if ∀ x , y ∈ A [x OP y ∈ A]
  • Ex. Natural Numbers (\(\mathbb{N}\))
    1. \(\mathbb{N}\) is closed under +
    2. \(\mathbb{N}\) is closed under ×
    3. \(\mathbb{N}\) is not closed under -
    4. \(\mathbb{N}\) is not closed under \
  • The class of all languages that are recognized by DFAs is closed under ∪

Closure Properties of DFAs

  • Union (∪)
  • Intersection (∩)
  • Complement (¬)
  • Reverse

Applying Closure Properties

  • If L(M1) ∪ L(M2) are DFAs, then ∃ DFA, M, with L(M) = L(M1) ∪ L(M2)
  • The purpose of a state machine is to make a yes/no decision

Premise:

  • M1 = {Q1,Σ,δ1,q01,F1} and M2 = {Q2,Σ,δ2,q0,F2} are DFAs
  • M1 accepts binary strings ending in 1
  • M2 accepts binary strings of odd length
  • L(M) = L(M1) ∪ L(M2)
    • Accepts binary strings that either end in 1 OR have odd length (or both)
  • M1: q1, q2 distinguished between ending in 0 and 1
  • M2: r1, r2 distinguished between odd and even length
  • Accordingly, M must be able to distinguish between:
    • even length ending in 1
    • even length ending in 0
    • odd length ending in 1
    • odd length ending in 0

Coding M

  • Consider Q = Q1 ⋅ Q2 = {q1r1,q2r1,q1r2,q2r2}
  • wherein
    • q1r1 = even string ending in 0
    • q1r2 = odd string ending in 0
    • q2r1 = even string ending in 1
    • q2r2 = odd string ending in 1
  • Ex. δ(q1r1,1) = q1r2
  • Applying this logic to a DFA, we know that
    • Q={q1r1,q1r2,q2r1,q2r2};
    • S={0,1};
    • d:Q \*sigma → Q;
    • d(q1r1,0)=q1r2;
    • d(q1r1,1)=q2r2;
    • d(q1r2,0)=q1r1;
    • d(q1r2,1)=q2r1;
    • d(q2r1,0)=q1r2;
    • d(q2r1,1)=q2r2;
    • d(q2r2,0)=q1r1;
    • d(q2r2,1)=q2r1;
    • q0=q1r1;
    • F={q1r2,q2r1,q2r2};

Derivation

  • Construct M and show that L(M) = L(M1) ∪ L(M2).
  • Q = Q1 ⋅ Q2
  • let qi ∈ Q1 and let rj ∈ Q2. and let c ∈ Σ
  • and qirj ∈ Q
  • thus, δ( qirj,c) = δ1(qi,c)δ2(rj,c)
  • and q0 = q01,q02
  • so that F={qirj: qi ∈ F1 ∪ rj ∈ F2}

Correctness

  • Show that M accepts exactly the strings that are accepted by M1 or M2
  • If w ∈ Σ*. tjem w is accepted by M and w is accepted by either M1 or M{2}
  • To do this, we can organize Σ* into strings of length 0, length 1, length 2…
  • We solve with mathematical induction, which is how we prove recurrence relationships

Mathematical Induction

  • Need base case and induction hypothesis
  • Induction hypothesis says something is true about k, where k is the length of strings
  • δ* (q0,w) is the ending state of M on w where
  • δ* (q0,w) = δ*1(q01,w)δ*2(q02,w)
  • Induction Hypothesis: If |w| = k, then δ*(q0,w) ∈ F \iff δ*1(q01,w) ∈ F1 or δ*2(q02,w) inf F2
  • Let x ∈ Σ.
  • Then |w ⋅ x| = k+1
  • And δ*(q0,wx) = δ*1(q01,wx)δ*2(q02,wx)
  • If δ*1(q01,wx) ∈ F1, δ*(q0,wx) ∈ F
  • Similarly, if delta*2(q02,wx), then δ*(q0,wx) ∈ F

Deterministic Finite Automata

  • We know that δ = Q × Σ → Q2
  • We want a function that takes a starting state and a string, then returns the state after the machine has read that string
  • Let’s define δ* = Q × Σ* → Q
    • δ* takes a state and a string
    • δ takes a state and a symbol
  • Now, we need a recursive definition
    • Base case:
      • Let qi ∈ Q
      • δ*(qi,ε) = qi
    • Recursive step:
      • If qi ∈ Q, w ∈ Σ*, and c ∈ Σ
      • then δ*(q, w ⋅ c) = δ(δ*(qi,w),c)

Aside: On Σ and Σ*

  • Σ* is the universe of all strings over Σ
    • Σ = {0,1}
    • Σ* = {ε,0,1,00,01,11,10,000,…}
  • We can see this recursively
    • Base step: ε ∈ Σ*
    • Recursive step:
    • Let w ∈ Σ*, let c ∈ Σ
    • Then w × c ∈ Σ*
  • We can see this recursion graphically
w c w ⋅ c step

Tuples & DFAs

  • The deterministic finite automaton shown is a 5-tuple:
    1. Q: finite nonempty set of states
      • state: configuration of logic of a machine
    2. Σ (Sigma) - input alphabet

    4. Q0 ∈ Q - starting state

    • δ: Q × Σ → Q2

Functions

  • Functions - something that maps objects from one set to another
  • Given f: a → b;
    • Everything in a is mapped to something in b
      • For every x, such that x is an element of a, there exists a y, such that y is an element of b

    • No one point in the domain can be mapped to two different points in the codomain

      • Logically, you can’t have a function that takes in one input and returns two different outputs
      • If f maps x → y1 and → y2, y1 = y2

      -∀ x ∈ A y1,y2 ∈ B [f(x)=y1 ∧ f(x)=y2 → y1 = y2]
